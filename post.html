> Introduction

Command-line tools are the backbone of developer productivity. When you run a CLI tool hundreds of times a day, every millisecond of startup time matters. Rust's zero-cost abstractions and minimal runtime make it an ideal choice for building snappy CLI applications.

In this post, we'll explore how to build a high-performance CLI tool that starts in under 10 milliseconds and processes millions of lines of input efficiently. We'll cover dependency management, async I/O patterns, and real-world optimization techniques.

[INFO]
This guide assumes basic familiarity with Rust. If you're new to the language, check out the official Rust Book first.

> Project Setup

Let's start by creating a new Rust project with the right dependencies:

bash 
cargo new fast-cli --bin
cd fast-cli

Update your Cargo.toml with essential dependencies:

toml 
[package]
name = "fast-cli"
version = "0.1.0"
edition = "2021"

[dependencies]
clap = { version = "4.4", features = ["derive"] }
tokio = { version = "1.35", features = ["full"] }
anyhow = "1.0"
serde = { version = "1.0", features = ["derive"] }

[profile.release]
opt-level = 3
lto = true
codegen-units = 1
strip = true
[WARNING]
The release profile settings above optimize for binary size and performance but will increase compilation time. Use them only for production builds.

> Performance Optimization

// Memory Allocation Strategy

One of the biggest performance wins comes from reducing allocations. Here's a comparison of different approaches:

rust 
// ❌ Slow: Allocates new String each iteration
fn process_lines_slow(input: &str) -> Vec<String> {
    input.lines()
        .map(|line| line.to_uppercase())
        .collect()
}

// ✅ Fast: Reuses buffer
fn process_lines_fast(input: &str) -> Vec<String> {
    let mut results = Vec::with_capacity(input.lines().count());
    let mut buffer = String::new();

    for line in input.lines() {
        buffer.clear();
        buffer.push_str(line);
        buffer.make_ascii_uppercase();
        results.push(buffer.clone());
    }
    results
}

> Real-World Example

Here's a complete example of a fast file processing CLI:

rust 
use clap::Parser;
use std::io::{self, BufRead, BufReader};
use std::fs::File;

#[derive(Parser)]
#[command(name = "fast-grep")]
#[command(about = "A blazingly fast grep alternative")]
struct Cli {
    pattern: String,
    file: String,

    #[arg(short, long)]
    case_insensitive: bool,
}

fn main() -> anyhow::Result<()> {
    let args = Cli::parse();
    let file = File::open(&args.file)?;
    let reader = BufReader::new(file);

    for (idx, line) in reader.lines().enumerate() {
        let line = line?;
        if matches_pattern(&line, &args.pattern, args.case_insensitive) {
            println!("{}:{}", idx + 1, line);
        }
    }

    Ok(())
}

fn matches_pattern(line: &str, pattern: &str, case_insensitive: bool) -> bool {
    if case_insensitive {
        line.to_lowercase().contains(&pattern.to_lowercase())
    } else {
        line.contains(pattern)
    }
}

> Conclusion

Building fast CLI tools in Rust requires attention to detail, but the language's zero-cost abstractions make it possible to achieve both ergonomic APIs and exceptional performance.

Key takeaways:

  • Minimize allocations by reusing buffers
  • Use buffered I/O for file operations
  • Leverage Rust's type system for zero-cost abstractions
  • Profile before optimizing - measure, don't guess
  • Consider parallel processing for CPU-bound tasks
[TIP]
For more advanced optimization techniques, check out the Rust Performance Book by Nicholas Nethercote.
